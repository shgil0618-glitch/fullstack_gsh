
# 🏦 BankSystem 1 - Control Version

> 반복문과 조건문만을 사용해 구현한  
> **콘솔 기반 사용자 참여형 뱅크 시스템**

---

## 📅 프로젝트 개요

- **제작일**: 2025-09-16  
- **구현 목적**

Java 기초 문법인 조건문과 반복문을 활용하여  
**사용자 입력에 따라 상태가 변화하는 프로그램 흐름**을  
직접 설계하고 구현하는 데 목적을 둔 프로젝트입니다.

회원가입, 로그인, 기능 선택 등  
모든 흐름을 변수와 제어문만으로 관리하며  
**절차형 프로그래밍의 한계와 구조적 사고의 중요성**을  
체감하는 것을 목표로 했습니다.

- **Language**: Java  
- **Input**: `java.util.Scanner`

- 조건문 (`if / else`, `switch`)

- 사용자 입력 처리

- 조건문과 반복문만을 사용한 **절차형 구조**

## 🎮 주요 기능

### 1️⃣ 메뉴 기반 상태 제어

- `level` 변수를 중심으로 한  
  **switch-case 구조의 메뉴 흐름 제어**
- 회원가입 → 로그인 → 기능 접근 순서를 강제하여  
  **상태 의존 로직** 구현



### 2️⃣ 회원 시스템

- 회원가입 / 로그인 / 계정 삭제 기능 제공  
- 로그인 여부에 따라 기능 접근 제한  
- 이전에 사용한 ID 재사용 방지 처리



### 3️⃣ 입금 / 출금 기능

- 잔액 검사 및 예외 처리  
- 입금 및 출금 시 경험치(`exp`) 증가  
- 잔액에 따른 **고객 등급 자동 판별**
  - Bronze / Silver / Gold / Diamond

단순 은행 시스템에서 벗어나  
**사용자 참여형 요소**를 추가했습니다.

  - “묻고 더블로 가!” 선택형 이벤트 구현

## 🧯 Trouble Shooting

### 🔴 문제: 메뉴 흐름 꼬임으로 인한 의도치 않은 기능 접근

초기 구현 단계에서
회원가입 또는 로그인 이전에도
입금, 출금, 조회 기능이 실행되는 문제가 발생했습니다.

이는 메뉴 선택 값만으로 기능을 분기하고,
**사용자 상태(회원가입 여부, 로그인 여부)를
명확히 구분하지 않았기 때문**이었습니다.

---

### 🛠 해결 방법

다음과 같은 **상태 관리 변수**를 도입해 문제를 해결했습니다.

* `add` : 회원가입 완료 여부
* `login` : 로그인 성공 여부
* `level` : 현재 메뉴 상태
* `old_level` : 로그인 이후 복귀할 메뉴 상태

각 기능 진입 전

```text
회원가입 여부 → 로그인 여부 → 기능 실행
```

순서를 강제하여
의도치 않은 흐름을 차단했습니다.
