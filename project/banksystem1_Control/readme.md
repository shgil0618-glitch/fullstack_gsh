## 🎮 시스템 특징

### 1️⃣ 메뉴 기반 상태 제어
- `level` 변수를 중심으로 **switch-case 구조**를 활용한 메뉴 흐름 제어
- 회원가입 → 로그인 → 기능 접근 순서를 강제하여 **상태 의존 로직** 구현

### 2️⃣ 회원 시스템
- 회원가입 / 로그인 / 계정 삭제 기능 제공
- 로그인 여부에 따라 기능 접근 제한
- 이전에 사용한 ID 재사용 방지 처리

## 🧯 Trouble Shooting

### 🔴 문제: 메뉴 흐름이 꼬이며 의도치 않은 기능 접근 발생

초기 구현 단계에서  
회원가입 또는 로그인 이전에도  
입금, 출금, 조회 기능이 실행되는 문제가 발생했다.

원인은 메뉴 선택 값만으로 기능을 분기하고,  
**사용자 상태(회원가입 여부, 로그인 여부)를 명확히 구분하지 않았기 때문**이었다.

---

### 🛠 해결 방법

다음과 같은 **상태 관리 변수를 도입**하여 문제를 해결했다.

- `add` : 회원가입 완료 여부
- `login` : 로그인 성공 여부
- `level` : 현재 메뉴 상태
- `old_level` : 로그인 이후 복귀할 메뉴 상태

각 기능 진입 시  
```text
회원가입 여부 → 로그인 여부 → 기능 실행
```

---

### ✅ 결과

- 회원가입/로그인 이전 기능 접근 문제 해결  
- 메뉴 흐름 안정화 및 의도한 순서대로 기능 수행  
- 사용자 상태 기반 제어 구조 확보 → 이후 **배열/메서드 버전 확장**에 활용 가능
